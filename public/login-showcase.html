pb-login

The PB login component encapsulates the authentication complexity. It comes with a button triggering the login procedure and a label displaying the authenticated user's name after a successful login. The button toggles to a 'logout' display as well. Logout invalidates the authentication, removes the user's name display and shows the 'login' display again. The button display text is localizable and adoptable on instance level. In the background pb-login maintains data structures for the back end calls of other parts of the application that is injects HTTP headers to the backend calls. pb-login supports implicit authentication. Without hitting the button explicitly it intercepts HTTP requests requiring authentication to ensure they are executed in a proper context. pb-login comes in 2 flavours: legacy-support (default false): true: Perform an authentication against auth.performancebuildings.ch. false: A makeshift BASIC authentication dialog for use cases within the PB admin VPNs or for test environments. hidden (default false): Don't show the login component. Fire the authentication immediately after page load.

Future enhancements

Support different keycloak configurations. Support other authentication mechanisms of customer eco-systems, e.g. LDAP, Active Directory, Liferay.
